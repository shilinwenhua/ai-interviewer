## 3. 组件设计

### 3.1 InterviewFlow (主容器)

**职责**：管理面试流程状态、协调子组件

```
Props:
- 无（内部管理状态）

State:
- step: 当前步骤
- role: 选择的角色
- company: 选择的公司
- position: 选择的职位
- questions: 题目列表
- answers: 答案列表
- result: 面试结果
```

### 3.2 RoleSelect

**职责**：展示角色选择卡片

```
Props:
- selected: 当前选中角色
- onSelect: 选中回调

UI:
- 4 个角色卡片（PM/前端/后端/运营）
- 选中高亮效果
```

### 3.3 CompanySelect

**职责**：公司 + 职位联动选择

```
Props:
- selectedCompany: 当前公司
- selectedPosition: 当前职位
- onSelect: 选择回调

UI:
- 公司列表（Tab 切换：国内 / 外企）
- 职位下拉框
- 确认按钮
```

### 3.4 JDPreview

**职责**：展示职位 JD

```
Props:
- company: 公司信息
- position: 职位信息

UI:
- 职位标题
- 薪资范围
- 岗位要求列表
- "开始面试" 按钮
```

### 3.5 QuestionPage

**职责**：答题页面

```
Props:
- question: 当前题目
- index: 当前题号
- total: 总题数
- answer: 用户答案
- onAnswerChange: 答案变化回调
- onSubmit: 提交回调
- onNext: 下一题
- onPrev: 上一题

UI:
- 题目内容
- 分类/难度标签
- 文本输入框
- AI 点评按钮（可选）
- 进度条
- 上下题按钮
```

### 3.6 InterviewResult

**职责**：面试反馈展示

```
Props:
- answers: 所有答案
- result: 面试结果

UI:
- 总分展示（环形进度）
- 五维雷达图
- 综合评语
- 改进建议列表
- 重新面试按钮
```

## 4. AI 点评逻辑

### 4.1 单题点评

调用硅基流动 API，返回：
```javascript
{
  score: 85,
  strengths: '回答结构清晰...',
  improvements: '建议补充数据...',
  suggestion: '可以这样回答...'
}
```

### 4.2 统一反馈生成

全部答题完成后，综合所有答案生成反馈：
```javascript
{
  专业技能: 85,
  项目经验: 80,
  逻辑思维: 75,
  沟通表达: 80,
  总分: 80,
  综合评语: '整体表现良好...',
  改进建议: [
    '建议加强数据分析能力',
    '项目描述可以更具体',
    '注意表达逻辑'
  ]
}
```

## 5. API 接口

### 5.1 单题点评

```javascript
POST https://api.siliconflow.cn/v1/chat/completions

{
  model: 'Qwen/Qwen2.5-72B-Instruct',
  messages: [{
    role: 'system',
    content: '你是一个专业面试官...'
  }, {
    role: 'user',
    content: `问题：${question}\n回答：${answer}`
  }]
}
```

### 5.2 生成反馈

```javascript
// 全部答题完成后，调用一次生成综合反馈
{
  messages: [{
    role: 'system',
    content: '你是一个专业面试官，请根据以下答案给出综合反馈...'
  }, {
    role: 'user',
    content: `角色：${role}\n公司：${company}\n职位：${position}\n答案：${JSON.stringify(answers)}`
  }]
}
```
