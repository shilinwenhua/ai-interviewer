# AI面试助手 V2.0 产品需求文档

**版本**：V2.0  
**类型**：功能迭代  
**日期**：2026-03-07  
**状态**：已确认

---

## 1. 版本目标

优化 AI 智能面试流程，还原真实面试场景，提供完整的面试反馈。

---

## 2. 面试流程

```
选择角色 → 选择公司/职位 → 查看JD → 开始答题 → 统一反馈
```

---

## 3. 功能需求

### 3.1 选择角色

| 功能 | 描述 |
|------|------|
| 角色选项 | 产品经理、前端工程师、后端工程师、运营专员 |
| 交互 | 点击卡片选择 |

### 3.2 选择公司/职位

| 功能 | 描述 |
|------|------|
| 公司列表 | 预设热门公司（国内大厂 + 外企） |
| 职位联动 | 选择公司后显示对应职位 |
| 数据源 | 本地预设 JD（暂不接入外部 API） |

### 3.3 公司/职位 JD 数据

#### 国内大厂

| 公司 | 职位 | JD 描述 |
|------|------|---------|
| 字节跳动 | 产品经理 | 负责字节跳动核心产品设计，要求具备数据驱动、增长思维 |
| 阿里 | 产品经理 | 阿里系产品，要求具备电商背景、平台思维 |
| 腾讯 | 产品经理 | 负责微信/QQ等核心产品，要求社交/内容经验 |
| 百度 | 产品经理 | AI 方向产品，要求懂 AI 技术、有搜索/推荐经验 |
| 美团 | 产品经理 | 本地生活产品，要求 O2O、履约产品经验 |
| 京东 | 产品经理 | 电商产品，要求供应链、物流经验 |
| 滴滴 | 产品经理 | 出行产品，要求出行、交易平台经验 |

#### 外企

| 公司 | 职位 | JD 描述 |
|------|------|---------|
| Meta | Product Manager | Global product, English required, data-driven |
| Google | Product Manager | Search/AI products, technical background preferred |
| Apple | Product Manager | Hardware+Software integration, design thinking |
| Amazon | Product Manager | AWS/EC2, customer obsession, data analytics |
| Microsoft | Product Manager | Cloud products, enterprise background |

### 3.4 查看 JD

| 功能 | 描述 |
|------|------|
| 展示内容 | 职位名称、公司、薪资范围、岗位要求 |
| 交互 | 可折叠/展开 |
| 开始 | 点击"开始面试"进入答题 |

### 3.5 开始答题

| 功能 | 描述 |
|------|------|
| 题库数量 | 8-10 道题 |
| 题目来源 | 根据角色选择对应题库 |
| 答题方式 | 文本输入 |
| AI 点评 | 每题可单独获取 AI 点评（可选） |
| 进度显示 | 当前第 X 题 / 共 Y 题 |
| 切换题 | 上一题/下一题 |

### 3.6 统一反馈（面试结束）

| 维度 | 描述 |
|------|------|
| 专业知识 | 0-100 分 |
| 项目经验 | 0-100 分 |
| 逻辑思维 | 0-100 分 |
| 沟通表达 | 0-100 分 |
| 总分 | 0-100 分 |
| 综合评语 | 200-300 字 |
| 改进建议 | 3-5 点 |

## 4. 页面结构

| 页面 | 路由 | 描述 |
|------|------|------|
| 角色选择 | /interview/role | 选择面试角色 |
| 公司选择 | /interview/company | 选择目标公司/职位 |
| JD 预览 | /interview/jd | 查看职位描述 |
| 答题页面 | /interview/start | 逐题答题 |
| 反馈页面 | /interview/result | 统一反馈 |
